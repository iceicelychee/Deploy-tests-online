## 项目说明
 本项目采用前后分离架构，前端负责页面展示和用户交互，后端提供数据接口，两者通过HTTP协议通信。

### 前端(VUE 3 + Options API)
  **技术栈**：VUE 3、Vue Router 、Axios

  **核心逻辑**：

    -页面组件(如`Users.vue`)在生命周期中通过Axios请求后端API获取数据。

    -用户操作(新增、删除)触发方法，通过Axios发生POST\DELETE请求，并根据后端相应更新前端状态(响应式数据)

    -子组件(`UserItem.vue`)通过`props`接收数据，通过`$emit`向父组件传递事件，实现职责分离。

### 后端(Node.js + Express)

 **技术栈**：Node.js、Express、CORS中间件

 **数据存储**：使用本地`users.json`文件持久化数据，通过`fs`模块读写。

  **API设计** (RESTUL风格)

    -`GET/users`：返回所有用户(从JSON文件读取)
    -`POST/users`：新增用户(接收JSON,追加数组并写回文件)
    -`DELETE/users/:id`：删除指定用户(从数组中移除，并写回文件)

### 数据流示意
1.用户访问`/users`页面 -> 前端请求`GET/users` -> 后端读取 `user.json`返回 -> 前端渲染列表。

2.用户点击“删除” -> 前端发送`DELETE/users/:id` -> 后端更新文件，返回204 -> 前端过滤列表。

3.用户提交新增表单 -> 前端发送`POST/users` -> 后端追加用户，写回文件，返回201和新数据 -> 前端将新用户加入列表。

### 关于 JSON 文件持久化与并发风险

当前使用 JSON 文件存储数据，优点是简单、无需数据库；但在高并发场景下(多个请求同时读写文件)可能会出现数据覆盖或丢失。本项目为学习用途，暂未引入文件或数据库，若生成需要可用，建议改用数据库或引入并发控制机制(读写锁、队列)
